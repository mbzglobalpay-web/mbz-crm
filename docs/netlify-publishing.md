# Публикация CRM МБЗ на Netlify

Цель: опубликовать CRM МБЗ так, чтобы она открывалась по ссылке:

```text
https://crm-mbz.netlify.app
```

и затем по домену:

```text
https://crm.mbz-bentonit.ru
```

CRM является статическим сайтом. Для публикации не нужны backend, PostgreSQL, API, 1C, ЭДО или интеграции.

## 1. Что Нужно Загрузить На Netlify

Загружать нужно корневую папку проекта целиком.

Минимальный набор файлов и папок:

```text
index.html
app/
netlify.toml
vercel.json
.nojekyll
README.md
docs/
import_templates/
screenshots/
```

Критически важные файлы для работы CRM:

```text
index.html
app/index.html
app/styles.css
app/app.js
app/assets/mbz-logo.svg
netlify.toml
```

Если загружать только папку `app/`, сайт тоже может открыться, но правильнее загружать весь корень проекта. Тогда Netlify откроет корневой `index.html`, а он перенаправит пользователя в CRM.

## 2. Подготовка Перед Публикацией

1. Откройте папку проекта CRM МБЗ.
2. Убедитесь, что внутри есть файл `index.html`.
3. Убедитесь, что внутри есть папка `app`.
4. Убедитесь, что внутри есть файл `netlify.toml`.
5. Убедитесь, что внутри `app` есть:
   - `index.html`;
   - `styles.css`;
   - `app.js`;
   - папка `assets`.
6. Не переименовывайте папку `app`.
7. Не переносите файлы из `app` в другие папки.
8. Не удаляйте `netlify.toml`.

## 3. Самый Простой Способ: Netlify Drop

Этот способ подходит, если вы новичок и пока не хотите связывать сайт с GitHub.

### Шаг 1. Открыть Netlify

1. Откройте браузер.
2. Перейдите на сайт:

```text
https://app.netlify.com/drop
```

3. Войдите в аккаунт Netlify.
4. Если аккаунта нет, создайте его.

Официальная документация Netlify описывает Netlify Drop как способ опубликовать папку сайта через drag-and-drop: [Netlify Drop](https://docs.netlify.com/start/get-started-with-drop/).

### Шаг 2. Загрузить CRM

1. Откройте папку с проектом CRM МБЗ на компьютере.
2. Выберите всю папку проекта, а не только `app`.
3. Перетащите папку проекта в окно Netlify Drop.
4. Дождитесь окончания загрузки.
5. Netlify создаст сайт и покажет временную ссылку вида:

```text
https://random-name-12345.netlify.app
```

### Шаг 3. Проверить Сайт

1. Нажмите на ссылку, которую выдал Netlify.
2. Убедитесь, что открылась CRM МБЗ.
3. Проверьте:
   - логотип МБЗ;
   - панель ГД;
   - меню слева или сверху на телефоне;
   - раздел "Первые 5 заказов";
   - раздел "Клиенты ГНБ";
   - раздел "Объекты ГНБ";
   - раздел "Образцы".

## 4. Как Получить Ссылку `crm-mbz.netlify.app`

Netlify по умолчанию выдает ссылку вида:

```text
https://имя-сайта.netlify.app
```

По документации Netlify, имя сайта можно изменить, и от этого изменится стандартный поддомен Netlify: [Default Netlify subdomain](https://docs.netlify.com/domains/domains-fundamentals/understand-domains/).

### Шаг 1. Открыть Настройки Сайта

1. Войдите в Netlify.
2. Откройте созданный сайт CRM.
3. В левом меню выберите `Site configuration`.
4. Откройте раздел `General`.
5. Найдите блок с названием сайта.

### Шаг 2. Переименовать Сайт

1. Нажмите `Change site name`.
2. Введите:

```text
crm-mbz
```

3. Нажмите `Save`.

### Шаг 3. Проверить Ссылку

Откройте:

```text
https://crm-mbz.netlify.app
```

Если имя `crm-mbz` уже занято, Netlify не даст сохранить его. Тогда выберите похожий вариант:

```text
crm-mbz-bentonit
crm-mbz-hdd
mbz-crm
```

После сохранения ссылка будет:

```text
https://выбранное-имя.netlify.app
```

## 5. Как Подключить Домен `crm.mbz-bentonit.ru`

Есть два способа:

- оставить DNS у текущего регистратора домена;
- перенести DNS-зону в Netlify DNS.

Для новичка проще первый способ: оставить DNS у текущего регистратора и добавить CNAME-запись.

### Способ A. DNS Остается У Текущего Регистратора

Используйте этот способ, если домен `mbz-bentonit.ru` уже куплен и управляется у регистратора или хостинг-провайдера.

### Шаг 1. Добавить Домен В Netlify

1. Откройте сайт CRM в Netlify.
2. Перейдите в `Site configuration`.
3. Откройте `Domain management`.
4. Найдите блок `Production domains`.
5. Нажмите `Add a domain`.
6. Введите:

```text
crm.mbz-bentonit.ru
```

7. Подтвердите добавление.

Netlify поддерживает custom domains и domain aliases для production-сайтов: [Netlify custom domains](https://docs.netlify.com/domains/domains-fundamentals/understand-domains/).

### Шаг 2. Добавить DNS-Запись У Регистратора

1. Откройте панель управления доменом `mbz-bentonit.ru`.
2. Найдите раздел DNS-записей.
3. Добавьте запись:

```text
Тип: CNAME
Имя / Host: crm
Значение / Target: crm-mbz.netlify.app
TTL: 300 или Auto
```

Если Netlify-ссылка отличается, например `crm-mbz-hdd.netlify.app`, укажите именно ее.

### Шаг 3. Дождаться DNS

1. DNS может обновиться за несколько минут.
2. Иногда требуется до 24 часов.
3. В Netlify рядом с доменом должен исчезнуть статус ожидания DNS.

### Шаг 4. Проверить HTTPS

1. В Netlify откройте `Domain management`.
2. Проверьте блок HTTPS / SSL.
3. Netlify обычно выпускает SSL-сертификат автоматически.
4. После выпуска сертификата откройте:

```text
https://crm.mbz-bentonit.ru
```

### Способ B. Перенести DNS В Netlify DNS

Используйте этот способ только если вы уверены, что можете менять NS-серверы домена.

1. В Netlify откройте раздел `Domains`.
2. Добавьте домен `mbz-bentonit.ru`.
3. Netlify покажет свои name servers.
4. У регистратора домена замените NS-серверы на Netlify.
5. После делегирования добавьте поддомен `crm.mbz-bentonit.ru` к сайту CRM.

Netlify описывает подключение существующего домена к Netlify DNS здесь: [Bring a domain to Netlify DNS](https://docs.netlify.com/domains/configure-domains/bring-a-domain-to-netlify/).

## 6. Как Обновлять CRM После Первой Публикации

Если сайт опубликован через Netlify Drop:

1. Внесите изменения в проект локально.
2. Проверьте, что CRM открывается.
3. Откройте сайт CRM в Netlify.
4. Перейдите в `Deploys`.
5. Найдите область для drag-and-drop deploy.
6. Перетащите обновленную корневую папку проекта.
7. Дождитесь нового deploy.
8. Ссылка останется прежней.

Netlify поддерживает обновление drag-and-drop сайта через новую загрузку папки на странице Deploys: [Create deploys](https://docs.netlify.com/deploy/create-deploys/).

## 7. Как Установить CRM На iPhone

Это будет ярлык CRM на главном экране iPhone. Пользователь сможет открывать CRM как приложение.

### Шаг 1. Открыть CRM В Safari

1. Откройте Safari на iPhone.
2. Введите ссылку:

```text
https://crm.mbz-bentonit.ru
```

или временную ссылку:

```text
https://crm-mbz.netlify.app
```

3. Дождитесь загрузки CRM.

### Шаг 2. Добавить На Экран Домой

1. Нажмите кнопку `Поделиться`.
2. Прокрутите список действий вниз.
3. Нажмите `На экран «Домой»`.
4. Проверьте название, например:

```text
CRM МБЗ
```

5. Нажмите `Добавить`.

### Шаг 3. Проверить

1. Вернитесь на главный экран iPhone.
2. Найдите иконку `CRM МБЗ`.
3. Нажмите на нее.
4. CRM должна открыться по ссылке.

## 8. Как Установить CRM На Android

Это будет ярлык CRM на главном экране Android.

### Шаг 1. Открыть CRM В Chrome

1. Откройте Chrome на Android.
2. Введите:

```text
https://crm.mbz-bentonit.ru
```

или:

```text
https://crm-mbz.netlify.app
```

3. Дождитесь загрузки CRM.

### Шаг 2. Добавить На Главный Экран

1. Нажмите меню Chrome с тремя точками.
2. Выберите `Добавить на главный экран`.
3. Если Chrome показывает `Установить приложение`, можно выбрать этот пункт.
4. Укажите название:

```text
CRM МБЗ
```

5. Нажмите `Добавить`.

### Шаг 3. Проверить

1. Вернитесь на главный экран Android.
2. Найдите иконку `CRM МБЗ`.
3. Откройте ее.
4. CRM должна загрузиться.

## 9. Как Давать Доступ Сотрудникам По Ссылке

После публикации доступ дается простой ссылкой.

### Вариант 1. Временная Ссылка Netlify

Отправьте сотрудникам:

```text
https://crm-mbz.netlify.app
```

### Вариант 2. Корпоративный Домен

После подключения домена отправьте:

```text
https://crm.mbz-bentonit.ru
```

### Что Написать Сотрудникам

Можно отправить такой текст:

```text
Коллеги, CRM МБЗ доступна по ссылке:
https://crm.mbz-bentonit.ru

Откройте ссылку в браузере.
На телефоне можно добавить CRM на главный экран:
- iPhone: Safari -> Поделиться -> На экран «Домой».
- Android: Chrome -> меню ⋮ -> Добавить на главный экран.
```

## 10. Важное Предупреждение По Безопасности

Текущая версия CRM является статическим прототипом и хранит данные в браузере пользователя.

Не размещайте в публичной версии:

- реальные коммерческие тайны;
- персональные данные без согласования;
- договоры;
- закрытые цены;
- внутренние финансовые документы.

Если нужна закрытая CRM с логином и правами доступа, это отдельный этап: backend, авторизация, база данных и разграничение ролей.
