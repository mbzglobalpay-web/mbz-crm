# CRM МБЗ для рынка ГНБ России

Статический прототип CRM для ООО "Мостовской Бентонитовый Завод" с фокусом на первые 5 заказов на рынке горизонтально-направленного бурения.

## Статус

Проект подготовлен как обычный статический local-first сайт. Его можно открыть:

- локально через `app/index.html`;
- через локальный web-сервер;
- после публикации на GitHub Pages, Netlify или Vercel.

Backend, PostgreSQL, API и интеграции не требуются.

## Рабочие Возможности

- создание клиента;
- редактирование клиента;
- удаление клиента со связанными записями;
- создание объекта ГНБ;
- создание КП;
- создание образца;
- создание испытания;
- создание заказа;
- сохранение данных в `localStorage`;
- экспорт данных в Excel;
- импорт данных из Excel.

## Структура

- `index.html` - корневой вход для хостинга, перенаправляет в CRM.
- `app/` - сама CRM.
- `app/index.html` - главный экран приложения.
- `app/styles.css` - фирменный стиль и адаптивная верстка.
- `app/app.js` - клиентская логика и демо-данные.
- `app/assets/mbz-logo.svg` - логотип МБЗ.
- `import_templates/` - шаблоны загрузки реальных данных.
- `docs/deployment.md` - инструкция публикации.
- `netlify.toml` - конфигурация Netlify.
- `vercel.json` - конфигурация Vercel.
- `.nojekyll` - отключение Jekyll для GitHub Pages.

## Локальный запуск

Вариант 1: открыть файл `app/index.html` в браузере.

Вариант 2: запустить любой статический сервер из корня проекта:

```bash
python -m http.server 4173
```

После этого открыть:

```text
http://localhost:4173/
```

## Публикация

Подробная инструкция:

[docs/deployment.md](docs/deployment.md)

Пошаговая инструкция именно для Netlify:

[docs/netlify-publishing.md](docs/netlify-publishing.md)

Коротко:

- GitHub Pages: опубликовать корень репозитория.
- Netlify: выбрать папку публикации `.`.
- Vercel: импортировать репозиторий как static project без build-команды.

## Проверка

Откройте опубликованную ссылку на ПК и телефоне. CRM должна загрузиться с корня сайта и перейти в `app/`.
